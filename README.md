## Контрольні питання
* **1 Це таке сховище або просто папка з кодом, де зберігається не тільки остання версія, а й вся історія правок.**
* **2 Щоб тримати код у порядку, не перекидати папки через Телеграм і працювати над одним проектом разом з іншими розробниками.**
* **3 Це софт, який робить «знімки» твого коду. Наприклад, Git — він стежить, що ти видалив або додав у файли.**
* **4 Щоб не боятися щось зламати: завжди можна відкотити код до моменту, коли все працювало, і бачити, хто саме з команди накосячив.**
* **5 Відкриваєш термінал у папці з проектом і пишеш git init. Все, Git почав за нею стежити.**
* **6 Прив'язуєш посилання на GitHub через remote add, а потім просто пушиш зміни (git push) на сервер або забираєш їх звідти через pull.**
* **7 Пишеш git branch і назву гілки. Це як паралельна реальність для коду — можна пиляти нову фічу, не ламаючи основну робочу версію.**
* **8 Це два кроки: спочатку кажеш Гіту, які файли береш (git add .), а потім «фоткаєш» їх командою git commit з коротким коментом.**
